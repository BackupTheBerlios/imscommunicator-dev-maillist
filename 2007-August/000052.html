Hi,

i just updated ims-communicator to JAIN-SIP API and RI 1.2 and found a 
implementation error regarding the via header:

RFC 3261, part 8.1.1.7 says, that a new branch id must be generated for 
each transaction.

This is not done in the ims-communicator as it uses the same local via 
header for all requests when it was generated once.
This caused some errors ("Transaction already exists") with the new 
JAIN-SIP stack.

The error occurs twice:
     sipManager.getLocalViaHeaders: here at least a new branch id has to 
be generated.
    RegisterProcessing.processAuthenticationChallenge: here also the old 
via header is used including the old branch id.

I think this is no problem with the old JAIN-SIP API 1.1, but the new 
one does not like it...
